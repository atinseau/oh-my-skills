# Setup — Prerequisites & Config

Run this before starting any mode. Skip steps already verified in this session.

## 1. Check playwright-cli binary

```bash
playwright-cli --version
```

- If the command exists → go to step 2
- If "command not found" → install it:
  ```bash
  npm install -g @playwright/cli@latest
  ```
  Then verify: `playwright-cli --version`

## 2. Check playwright-cli skill

The `playwright-cli` skill must be available (it provides the full command reference for browser automation).

- If the skill is listed in available skills → go to step 3
- If the skill is not available → install it:
  ```bash
  playwright-cli install --skills
  ```
  This installs the skill files into the `.claude/skills/` directory (or `.agents/skills/` depending on the project).

## 3. Load or create config.yaml

### If config.yaml exists

1. Read `.discovery/config.yaml`
2. Read `.discovery/scenarios/_index.md` — understand current coverage
3. If the user gives a specific task → go to that mode
4. If the user says "continue" or "what's next?" → suggest the next step based on coverage gaps
5. Default mode is **explore** — if unsure, explore

### If no config.yaml (first run)

1. Read `playwright.config.ts` (or `.js`) → extract `webServer.command`, `webServer.url`, `testDir`
2. Glob for existing patterns → `helpers/`, `mocks/` or `fixtures/`, `*.spec.*` files
3. Propose the detected values to the user for confirmation
4. Write `.discovery/config.yaml`
5. If a value cannot be detected → ask the user

### config.yaml format

```yaml
# Auto-generated by e2e-discovery — edit freely
discovery_root: tests/.discovery
dev_command: pnpm e2e:dev
base_url: http://localhost:5173
test_command: pnpm e2e
test_dir: tests/orchestrator
helpers_dir: tests/helpers
mocks_dir: tests/mocks/data
spec_suffix: .spec.ts
```
